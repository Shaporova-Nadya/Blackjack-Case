## Blackjack
*Приложение для игры в Blackjack*

# Архитектура приложения для игры в Blackjack

## Компоненты программы
- 'Card', 'Deck', 'Hand', 'Player', 'HumanPlayer', 'Dealer' - игровые сущности
- 'Suit', 'Rank', 'Action', 'GameState' - перечисления для фиксирванных значений
- 'Outcome' - перечисление возможных результатов раунда

### Логика игры
- 'Game' - отвечает за координацию раунда: начало раунда, раздача, ходы, завершение
- 'GameAdmin' - проверка корректности ходов

### Статистика и хранение
- 'Statistics' - отвечает за накопление данных раундов, вычисление статистики и рейтингов
- 'Database' - сохранение и загрузка

### Управление сессией
- 'Session' - жизненный цикл приложения: создание игры, запуск раундов, сохранение при выходе
- 'Main' - точка входа, создает 'Session'

## Сценарий игры

1. 'Main' создает 'Session'
2. 'Session' при помощи метода 'startNewSession()' создает 'Game', 'Statistics', 'Database'
3. 'Game' проводит раунды:
   - Игроки делают ставки
   - 'Deck' раздает карты
   - 'GameAdmin' проверяет допустимые действия
   - По завершению раунда 'Game' передает результат в 'Statistics.recordRound()'
4. 'Statistics' обновляет данные в 'RoundData'
5. При завершении сессии 'Session.exitGame()' сохраняет 'GameSnapshot' через 'Database', а после, закрывает приложение
